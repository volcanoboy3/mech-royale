# Project Structure

Mech Royale keeps the playable web game at the repository root so the existing
Vercel deployment continues to work. Native and generated files live in their
own folders.

| Path | Purpose |
| --- | --- |
| `index.html` | Main HTML Canvas game and mobile interface |
| `assets/` | Web artwork and terrain textures |
| `api/` | Vercel server functions |
| `supabase/` | Cloud-save database setup |
| `ios/` | Native iPhone project generated by Capacitor |
| `scripts/` | Repeatable web and iOS build scripts |
| `docs/` | Project, App Store, and release documentation |
| `dist/` | Generated local web bundle used by the iOS app (not committed) |

## Important Rules

- Edit the root `index.html`; do not edit the generated copy in `dist/`.
- Run `npm run build:web` before syncing the iOS project.
- Keep secrets out of the repository. Supabase public configuration is fetched
  from the Vercel API when available.
- Commit the native `ios/` project, but not generated web copies, Pods, or build
  output.
